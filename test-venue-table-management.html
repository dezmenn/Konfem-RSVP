🧪 Venue & Table Management Testing

User Testing Checkpoint - Task 13

🌐 Open Web App 📊 Venue API 🪑 Tables API

🎯 Testing Objectives

This checkpoint validates the venue layout and table management functionality, including:

  • Venue element creation, positioning, and management
  • Table creation, capacity management, and positioning
  • Drag-and-drop functionality for venue design
  • Table locking and unlocking features
  • Layout validation and conflict detection
  • Integrated venue and table management interface

🏛️ Venue Element Testing

Scenario 1: Element Library Usage

  • Open element library panel
  • Drag elements from library to canvas
  • Verify elements appear with correct default properties
  • Test different element types (stage, bar, dance floor, etc.)

Scenario 2: Element Manipulation

  • Select existing elements by clicking
  • Drag elements to new positions
  • Resize elements using corner handles
  • Edit element properties (name, color)
  • Duplicate and delete elements

🪑 Table Management Testing

Scenario 3: Table Creation and Configuration

  • Double-click canvas to create new tables
  • Edit table names and capacity
  • Verify capacity validation (positive numbers only)
  • Test table positioning via drag-and-drop

Scenario 4: Table Locking Features

  • Lock tables using the lock button
  • Verify locked tables show lock indicator
  • Test that locked tables resist auto-arrangement
  • Unlock tables and verify functionality

🔄 Integrated Testing

Scenario 5: Combined Layout Management

  • Switch between venue and table modes
  • Test zoom controls (+ / - buttons)
  • Verify both elements and tables are visible
  • Test layout validation for complete venue
  • Check capacity information panel
